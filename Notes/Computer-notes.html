Lesson 1.1: Networking and Telecommunication

Introduction

Telecommunication: The process of transmitting information over distances via electronic means (e.g., mobile phones, internet, radio, television).

Computer Network: An interconnection of computing devices via wired or wireless media to facilitate data exchange.

Developed nations used Electronic Data Processing Centers (EDP) (1950–1980) for data handling.

In Nepal:

  • National Computer Center (NCC) operated until 1996.
  • Microcomputer processed census data (1998).

Services like hotel booking, air ticketing, banking, email, etc., are enabled via computer networks.

Data Communication Modes

Mode Direction Characteristics Examples
Simplex One-way Unidirectional transmission TV, Radio, Books
Half-Duplex Two-way (one at a time) Data transmission in both directions but not simultaneously Walkie-Talkie, Wireless Handsets
Full-Duplex Two-way (simultaneous) Allows simultaneous bidirectional communication Mobile Phones, Landline Phones

Components of Data Communication

  • Data (Message): The information being transmitted.
  • Sender: The device that initiates data transmission.
  • Medium: The communication channel used (wired or wireless).
  • Receiver: The device that receives the transmitted data.
  • Protocol: A set of rules governing data transmission.

Communication Media

1. Guided (Wired) Media:

Type Description Speed & Distance Usage
Twisted Pair Cable Copper wires twisted together, used for reducing interference. Up to 100 Mbps, 100m distance Telephone connections, LAN
Coaxial Cable Contains an insulated conductor surrounded by a shield for better signal quality. Up to 10 Gbps, 500m distance Cable TV, Long-distance telephone lines
Fiber Optic Cable Transmits data using light signals, immune to electromagnetic interference. Up to several Tbps, 100km+ distance High-speed internet, ISP backbone

2. Unguided (Wireless) Media:

Type Description Usage
Radio Waves Used for long-range wireless transmission. AM/FM radio, Mobile communication
Microwave High-frequency waves requiring a clear line of sight. Mobile network towers, Satellite links
Satellite Communication Transmission via artificial satellites in space. Global Internet, GPS, TV broadcasting

Computer Networks

Definition: A group of interconnected computers that share resources and exchange data.

Advantages:

  • Hardware Sharing (e.g., printers, scanners).
  • Information Sharing (e.g., internet, emails).
  • Centralized Data Management.
  • Cost Efficiency.

Disadvantages:

  • Security Risks (e.g., viruses, malware, hacking).
  • High Setup Costs.
  • Cybercrime Possibilities.

Types of Computer Networks

Type Coverage Area Speed Usage Example
LAN (Local Area Network) Small (Room, School, Office) High-speed (up to 10 Gbps) Private networks for small areas School labs, Cybercafes
MAN (Metropolitan Area Network) City-wide or large area Moderate speed (up to 1 Gbps) Connects multiple LANs in a larger area Cable TV networks, Government networks
WAN (Wide Area Network) Global scale Variable speed (Kbps to Tbps) Connects multiple LANs/MANs globally The Internet

Network Architectures

Architecture Description Pros Cons
Peer-to-Peer Equal-status devices share resources directly Simple setup, Cost-efficient Less secure, Limited scalability
Client-Server Centralized server handles data requests Secure, Efficient management Higher costs, Requires maintenance
Centralized Main computer stores and processes all data Efficient for large-scale data handling Risk of failure if the central system crashes

Network Topologies

Topology Description Pros Cons
Bus Devices connect to a single backbone cable Cost-effective Failure in backbone disrupts network
Ring Devices form a circular connection Efficient data transfer If one node fails, the network stops
Star Devices connect via a central hub/switch Easy troubleshooting Hub failure crashes entire network
Mesh All nodes are connected to multiple other nodes High fault tolerance, Fast data transfer Expensive setup, Complex management

Internet & Its Services

Definition: A worldwide interconnection of networks using TCP/IP protocol.

History:

  • ARPANET (1969) – U.S. Department of Defense project leading to Internet evolution.
  • Nepal (1994) – Internet service began via Mercantile Communications.

Common Services:

  • World Wide Web (WWW) – Web documents via HTTP.
  • Email – Electronic mail communication.
  • Telnet – Remote login to host computers.
  • FTP (File Transfer Protocol) – Data exchange between computers.
  • Search Engines – Web searching tools (Google, Bing, Yahoo).
  • IRC (Internet Relay Chat) – Online chatting.
  • Online Banking – Financial transactions online.
  • E-Commerce – Buying & selling goods via the Internet.

Lesson 1.2: Ethical and Social Issues in ICT

Introduction

Ethics refers to principles guiding decisions that lead society or individuals toward a better life. In ICT, ethics focus on the proper and improper usage of digital technologies. With increased connectivity via the Internet and social networking, concerns regarding personal data security, online behavior, and digital responsibility arise. ICT ethics address issues like privacy, intellectual property, security, and digital citizenship.

Computer Ethics

Definition: A set of moral principles that regulate the use of computers.

Ethical concerns in computing:

  • Privacy Violation – Unauthorized access to personal or sensitive data.
  • Intellectual Property Theft – Copying or using software illegally.
  • Hacking – Unauthorized access to computing systems.
  • Cybercrime – Criminal activities conducted via computers.

Key Commandments of Computer Ethics:

  • Do not harm others using computers.
  • Do not steal or alter information unlawfully.
  • Do not use a computer to publish false information.
  • Do not interfere with someone else's work or records.
  • Do not copy software without permission.

Cyber Ethics

Definition: A code of conduct for using the Internet responsibly.

Cyber ethics covers:

  • Proper communication online (no cyberbullying, hate speech).
  • Respect for privacy (not sharing personal data of others).
  • Protection of intellectual property (avoiding piracy).
Do’s Don’ts ✅ Use the Internet for research and learning. ❌ Do not copy data from the internet. ✅ Secure your personal data with strong passwords. ❌ Do not share personal information with strangers. ✅ Verify information before sharing it. ❌ Do not post or share hate speech. ✅ Respect intellectual property rights. ❌ Do not engage in cyberbullying. ✅ Follow terms and conditions of online platforms. ❌ Do not access unauthorized systems. ✅ Use antivirus software and firewalls. ❌ Do not use the internet to harm others. ✅ Report suspicious activities or content. ❌ Do not spread misinformation.

Threats in Social Media

Benefits:

Benefit Description Branding Businesses can promote products and engage customers. Communication Connects people across the world instantly. Learning & Collaboration Enables online education and professional networking.

Threats:

Threat Description Fake Accounts Used for fraud and identity theft. Phishing Attacks Trick users into revealing sensitive information. Misinformation False news spreads rapidly. Cybercrime Online fraud, hacking, data breaches.

Lesson 1.3: Computer Security

Introduction

Computer security refers to protecting computer systems, data, and networks from unauthorized access, damage, theft, or cyber threats. As computers are interconnected via the internet, threats like hacking, malware, and data breaches have increased. Computer security ensures:

  • Confidentiality (protects sensitive data).
  • Integrity (prevents unauthorized modifications).
  • Availability (keeps systems operational).

Information Security

Protects stored or transmitted data from loss or unauthorized access.

Causes of Data Loss:

  • Accidental deletion.
  • Hard disk failure.
  • Theft or natural disasters (fire, flood, etc.).

Preventive Measures:

  • Regular backups (external drive, cloud storage).
  • Secure storage (password-protected files).
  • Data encryption (protects sensitive files from unauthorized access).

Cybersecurity Threats

Threat Description Impact
Phishing Fake messages tricking users into revealing data. Identity theft, financial loss.
Botnet A network of infected devices controlled remotely. Massive cyberattacks.
Rootkit Malicious software that hides its presence. Full system takeover by hackers.
Keylogger Tracks and records keyboard inputs. Stolen passwords and sensitive information.
Hacker Person who gains unauthorized access to systems. Data breaches, cybercrime.
Drive-by Attack Malware installs without user action. System infection.

Security Mechanisms

Authentication Systems

Ensures authorized access using credentials like:

  • Password – Must be strong and regularly updated.
  • Biometric Authentication – Uses fingerprints, facial recognition, retina scans.
  • Two-Factor Authentication (2FA) – Requires additional verification (code via SMS/email).
  • Firewalls – Monitor and control incoming/outgoing network traffic to prevent unauthorized access.
  • Cryptography – Secures data using encryption methods.

Encryption & Decryption

Process Definition Purpose
Encryption Converts data into unreadable format. Protects sensitive information.
Decryption Converts encrypted data back to readable form. Allows authorized users to access it.

Types of Malware

Type Description Impact
Virus Self-replicating malicious program. Corrupts files and systems.
Spyware Secretly collects user data. Privacy invasion, identity theft.
Trojan Disguises as a legitimate program. Creates security vulnerabilities.
Worm Spreads through networks without user action. Slows down system performance.
Adware Displays unwanted advertisements. Disrupts user experience.

Antivirus Software

Detects and removes viruses, malware, spyware from systems.

Examples of popular antivirus programs:

  • Bitdefender
  • Norton
  • McAfee
  • Kaspersky

Importance of Updates:

  • Keeps virus definitions current.
  • Detects and eliminates new threats.

Backup System

Essential for data recovery in case of loss or corruption.

Types of backups:

  • Local Backup – External hard drive, USB.
  • Cloud Backup – Online storage (Google Drive, Dropbox).
  • Restores data quickly in case of system failure.

Hardware Security

Threats to Hardware:

  • Dust & Dirt – Can damage components.
  • Water & Moisture – Causes short circuits.
  • Power Surges – Leads to overheating or failure.
  • Theft – Loss of computer systems and sensitive data.
  • Fire Damage – Destroys electronic circuits.

Preventive Measures:

Threat Solution
Dust Regular cleaning, air filters.
Power Fluctuations Use UPS, surge protectors.
Theft Lock systems, restricted access.
Fire Risk Install fire extinguishers.

Lesson 1.6: Number System

Introduction

A number system is a way of representing numbers using a set of digits or symbols. [cite: 94]

Computers process data using different number systems based on positional notation. [cite: 95]

The base (or radix) of a number system determines the total number of digits available. [cite: 96]

Types of Number Systems

Number System Base (Radix) Digits Used Example
Decimal 10 0–9 (539)₁₀
Binary 2 0, 1 (1010)₂
Octal 8 0–7 (120)₈
Hexadecimal 16 0–9, A–F (7A2)₁₆

Decimal Number System

Commonly used in daily life, consists of 10 digits (0–9). [cite: 99]

Example : (539)₁₀ = (5 × 10²) + (3 × 10¹) + (9 × 10⁰) [cite: 100]

Binary Number System

Used in computers and digital systems. [cite: 100]

Uses only two digits: 0 and 1. [cite: 101]

Example : (1011)₂ = (1 × 2³) + (0 × 2²) + (1 × 2¹) + (1 × 2⁰) = 11₁₀ [cite: 101]

Octal Number System

Used in microprocessors and memory addressing. [cite: 101]

Uses eight digits (0–7). [cite: 102]

Example : (120)₈ = (1 × 8²) + (2 × 8¹) + (0 × 8⁰) = 64 + 16 + 0 = 80₁₀ [cite: 102]

Hexadecimal Number System

Compact representation for binary numbers. [cite: 102]

Uses sixteen digits (0–9 and A–F). [cite: 103]

Example : (7A2)₁₆ = (7 × 16²) + (A × 16¹) + (2 × 16⁰) = (7 × 256) + (10 × 16) + (2 × 1) = 1954₁₀ [cite: 103]

Number System Conversions

Decimal to Other Number Systems

Decimal to Binary :

Divide by 2, record remainders, read remainders bottom to top. [cite: 104]

Example : (25)₁₀ → (11001)₂ [cite: 104]

Decimal to Octal :

Divide by 8, record remainders, read bottom to top. [cite: 105]

Example : (345)₁₀ → (531)₈ [cite: 105]

Decimal to Hexadecimal :

Divide by 16, record remainders, read bottom to top. [cite: 106]

Example : (88)₁₀ → (58)₁₆ [cite: 106]

Binary to Other Number Systems

Conversion Method
Binary to Decimal Multiply each binary digit by its place value (powers of 2). [cite: 107]
Binary to Octal Group three binary digits from right to left, convert each group to octal. [cite: 107]
Binary to Hexadecimal Group four binary digits from right to left, convert each group to hex. [cite: 107]

Examples

Binary to Decimal :

(110101)₂ → (1 × 2⁵) + (1 × 2⁴) + (0 × 2³) + (1 × 2²) + (0 × 2¹) + (1 × 2⁰) = 53₁₀ [cite: 108]

Binary to Octal :

(100010011000)₂ → Group into threes → (100) (010) (011) (000) → (4) (2) (3) (0)₈ [cite: 108]

Binary to Hexadecimal :

(110010011)₂ → Group into fours → (0001) (1001) (0011) → (1) (9) (3)₁₆ [cite: 108]

Octal & Hexadecimal Conversions

Conversion Method
Octal to Decimal Multiply digits by powers of 8. [cite: 109]
Hexadecimal to Decimal Multiply digits by powers of 16. [cite: 109]
Octal to Binary Convert each octal digit into three binary digits. [cite: 109]
Hexadecimal to Binary Convert each hex digit into four binary digits. [cite: 109]

Examples

Octal to Decimal :

(157)₈ → (1 × 8²) + (5 × 8¹) + (7 × 8⁰) = 111₁₀ [cite: 110]

Hexadecimal to Decimal :

(1AC)₁₆ → (1 × 256) + (A × 16) + (C × 1) = 428₁₀ [cite: 110]

Lesson 3.2: Modular Programming

Introduction

Modular Programming is a technique that divides a program into smaller, manageable, and functional modules or procedures.

These modules work independently, but together they achieve the full functionality of a program.

It follows a structured programming approach, improving readability, reusability, debugging, and maintenance.

QBASIC supports modular programming using:

  • Sub-procedures (SUB ... END SUB).
  • Function-procedures (FUNCTION ... END FUNCTION).

Advantages of Modular Programming

Advantage Description
Reduces Complexity Breaks large programs into smaller, understandable modules.
Reusability Modules can be reused in different parts of the program or other programs.
Easy Debugging Errors are isolated to individual modules.
Improved Maintainability Easier to update and modify specific parts of a program.
Faster Development Multiple programmers can work on different modules simultaneously.

Types of Procedures in QBASIC

Sub-Procedure

A sub-procedure performs specific tasks without returning a value.

Defined using SUB ... END SUB, and executed using CALL.

Example :

DECLARE SUB Greet ()
CLS
CALL Greet
END
SUB Greet
    PRINT "Hello, welcome to QBASIC Modular Programming!"
END SUB

Function-Procedure

A function-procedure performs calculations and returns a value to the calling module.

Defined using FUNCTION ... END FUNCTION, and called using variable assignment.

Example :

DECLARE FUNCTION AddNumbers (num1, num2)
CLS
INPUT "Enter first number: ", a
INPUT "Enter second number: ", b
sum = AddNumbers(a, b)
PRINT "Sum: ", sum
END
FUNCTION AddNumbers (num1, num2)
    AddNumbers = num1 + num2
END FUNCTION

Call by Value vs. Call by Reference

Aspect Call by Value Call by Reference
Definition Passes a copy of the argument. Passes the memory address of the argument.
Modification Original variable remains unchanged. Original variable can be modified.
Memory Usage Uses more memory for copies. Uses less memory (no copies).
Safety Safer, as original data is protected. Less safe, as original data can be altered.

Example (Call by Value):

DECLARE SUB ShowValue (val)
CLS
num = 10
PRINT "Before call: "; num
CALL ShowValue(num)
PRINT "After call: "; num
END
SUB ShowValue (val)
    val = val + 5
    PRINT "Inside sub: "; val
END SUB

💡 Output : Before call: 10 Inside sub: 15 After call: 10 (Original variable unchanged).

Example (Call by Reference):

DECLARE SUB AddValue (total)
CLS
SHARED total
total = 10
PRINT "Before call: "; total
CALL AddValue(total)
PRINT "After call: "; total
END
SUB AddValue (total)
    SHARED total
    total = total + 5
END SUB

💡 Output : Total: 15 (Changes apply globally).
